# /data-analytics

## 前置输入

读取 Practice Profile → 方法论 → 可用分析工具。

## 分析模式

| 参数 | 模式 | 适用场景 |
|------|------|---------|
| （默认）| 一次性分析 | 单一审计项目的定向数据分析 |
| `--continuous` | 持续监控 | 定期运行的分析规则集，发现异常自动预警 |
| `--refresh` | 刷新上次分析 | 沿用上次的分析参数，更新数据后重新运行 |

## 工作流

### Step 1：定义分析目标

```
分析目标：[描述要回答的审计问题]
数据源：[ERP / OA / 业务系统 / 用户上传]
数据期间：[YYYY-MM 至 YYYY-MM]
分析类型：[描述性 / 诊断性 / 预测性 / 规范性]
```

### Step 2：数据获取与质量检查

```
┌─ 数据质量检查 ─────────────────────────────────────────────┐
│ 检查项               │ 结果            │ 处理措施         │
│ 记录总数             │ 10,000 行       │ —               │
│ 缺失值               │ 2.3%           │ 标记后填充默认值  │
│ 重复记录             │ 15 条           │ 去重保留 1 条    │
│ 数据类型一致性        │ ✓              │ —               │
│ 时间范围完整性        │ ✓ 覆盖全年      │ —               │
│ 异常值               │ 3 个离群值       │ 单独标注不剔除   │
└──────────────────────────────────────────────────────────┘
```

### Step 3：运行分析规则（按需选择）

#### 规则类：合规性检查

```
┌─ 合规性规则检查 ─────────────────────────────────────────────┐
│ 规则                    │ 异常数 │ 异常率 │ 说明              │
│ 超审批权限的采购订单      │ 12     │ 0.8%  │ 部门经理签了超额度单 │
│ 缺少入库确认的付款       │ 5      │ 0.3%  │ 先付款后补入库      │
│ 同一供应商连续大额采购    │ 3      │ 0.2%  │ >¥50万/周         │
│ 员工费用报销缺少附件      │ 28     │ 5.6%  │ 须补充凭证         │
└──────────────────────────────────────────────────────────────┘
```

#### 趋势类：异常波动检测

```
┌─ 趋势异常检测 ───────────────────────────────────────────────┐
│ 指标                 │ 当期      │ 上期      │ 变动    │ 标记  │
│ 采购总额             │ ¥2,000万 │ ¥1,800万 │ +11%   │ ✓    │
│ 平均采购周期          │ 15 天     │ 12 天     │ +25%   │ ⚠    │
│ 单笔平均采购金额       │ ¥8.5万   │ ¥6.2万   │ +37%   │ ⚠    │
│ 紧急采购单占比         │ 12%      │ 8%        │ +50%   │ ⚠    │
│ 供应商数量            │ 85       │ 72        │ +18%   │ ✓    │
└──────────────────────────────────────────────────────────────┘
```

#### 关联类：多数据源交叉验证

```
┌─ 交叉验证规则 ───────────────────────────────────────────────┐
│ 验证                     │ 一致  │ 差异说明                │
│ HR 人员名单 vs OA 审批权限 │ ✓    │ —                      │
│ ERP 库存数 vs 盘点记录     │ ⚠   │ 差异 3 项，绝对值 ¥2.8 万 │
│ 采购订单金额 vs 合同金额   │ ⚠    │ 5 笔订单金额大于合同    │
│ 报销金额 vs 差旅标准       │ ⚠    │ 超标报销 8 笔，超总额 ¥1.2 万 │
└──────────────────────────────────────────────────────────────┘
```

#### 连续监控（`--continuous` 模式）

```
┌─ 连续审计监控规则 ───────────────────────────────────────────┐
│ 监控规则                     │ 阈值/频率 │ 最近一次检查 │ 状态  │
│ 超权限采购                    │ 每周     │ 2025-02-03  │ ✓    │
│ 未匹配付款                    │ 每日     │ 2025-02-03  │ ⚠ 2笔│
│ 大额非常规费用                │ 每周     │ 2025-02-03  │ ✓    │
│ 异常登录/系统访问              │ 每日     │ 2025-02-03  │ ✓    │
└──────────────────────────────────────────────────────────────┘
```

```

```

### Step 5：输出


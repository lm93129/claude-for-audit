# /report-drafting

## 重要：先加载 Practice Profile

读取 → 输出 → 报告风格（详细/简洁）、发现分类标准、报告分发范围。

## 前置要求

需已完成：
- ✅ 审计方案（`/internal-audit:audit-program`）
- ✅ 风险控制矩阵（`/internal-audit:risk-control-matrix`）
- ✅ 审计发现清单征求意见（`/internal-audit:finding-drafting`）
- ✅ 被审计单位对发现的书面确认或异议

如未完成 → 提示先完成上述步骤。

## 报告模板

### 报告头

```
┌─────────────────────────────────────────────────────────┐
│                   内部审计报告                               │
│                                                          │
│ 报告编号：[年份]-[项目编号]-[序号]                           │
│ 密级：内部 — 仅限授权人员                                    │
│                                                          │
│ 被审计单位：[部门/公司名称]                                    │
│ 审计类型：[运营/合规/财务内控/专项]                            │
│ 审计期间：[YYYY-MM 至 YYYY-MM]                             │
│ 报告日期：[YYYY-MM-DD]                                     │
│                                                          │
│ 审计组：[姓名、姓名]                                         │
│ 审计负责人：[姓名]                                          │
└─────────────────────────────────────────────────────────┘
```

### 执行摘要（`--executive` 模式下主要输出，`--detail`模式下为摘要）

```

审计期间为[YYYY年MM月至YYYY年MM月]。

### 总体评价
[总体审计意见]

### 审计发现汇总

  — [标题]

  — [标题]
🟡 改善建议：N 项
  — [标题]

### 关键数据

└─────────────────────────────────────────────────────────┘
```

### Step 3：征求意见与定稿

报告草稿 → 发送被审计单位征求意见（7-10 个工作日）→ 修改 → 定稿。

```
报告状态跟踪：
□ 初稿完成 [日期]
□ 发送征求意见 [日期]
□ 收到反馈 [日期]
□ 定稿完成 [日期]
□ 审计负责人签发 [日期]
□ 分发给：[审计委员会 / 管理层 / 被审计单位]
```

### Step 4：输出

```
[项目编号]_REPORT_v1.md — 内审报告（征求意见稿）
[项目编号]_REPORT_final.md — 内审报告（定稿）

报告分发清单：
  [ ] 审计委员会
  [ ] CEO/总经理
  [ ] CFO（如涉及财务）
  [ ] 被审计单位负责人
  [ ] 合规部门（如涉及合规事项）
```

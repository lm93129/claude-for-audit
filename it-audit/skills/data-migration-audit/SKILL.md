# /data-migration-audit

## 工作流

### Step 1：迁移概况

```
项目：[系统名称]
源系统：[名称]
目标系统：[名称]
迁移对象：[主数据/交易数据/余额/附件/日志]
切换窗口：[日期时间]
迁移策略：[一次性切换 / 分批 / 双轨运行]
```

### Step 2：关键检查点

| 阶段 | 检查点 |
|------|------|
| 迁移前 | 范围确认、字段映射、清洗规则、职责分工、回退方案 |
| 迁移中 | ETL 执行日志、失败记录、补数机制、控制总量 |
| 迁移后 | 条数/金额对账、关键字段抽样、接口恢复、业务验收 |

### Step 3：典型输出

```
对账清单：
- 客户主数据：源 12,500 条 / 目标 12,497 条 → ⚠ 缺 3 条
- 应收余额：源 ¥58,320,000 / 目标 ¥58,320,000 → ✓
- 发票附件：源 4,200 个 / 目标 4,050 个 → ⚠ 缺 150 个
```

### Step 4：输出

```
[项目编号]_MIG_[日期].md — 数据迁移审计报告
├── 项目背景与切换策略
├── 迁移控制检查结果
├── 对账与抽样测试
├── 缺陷与补救建议
└── 是否建议正式切换/关闭缺陷
```

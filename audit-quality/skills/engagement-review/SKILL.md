# /engagement-review

## 核心检查

### Step 1：项目概况

```
客户：[名称]
项目级别：[一般 / 重点 / 高风险]
报告意见类型：[无保留/保留/否定/无法表示]
关键审计事项：[列表]
```

### Step 2：复核重点

| 模块 | 检查重点 |
|------|---------|
| 风险评估 | 识别的重大错报风险是否与后续程序一致 |
| 重要性 | 计算和运用是否前后一致 |
| 重大判断 | 会计估计、持续经营、错报评价是否有充分记录 |
| 审计证据 | 是否充分、适当、可追溯 |
| 报告结论 | 是否与底稿证据一致 |
| KAM | 关键审计事项的选择和描述是否适当 |

### Step 3：输出

```
[项目编号]_EQR_[日期].md — 项目质量复核记录
├── 项目背景
├── 复核重点与结论
├── 要整改事项
└── 复核意见（通过/有条件通过/不通过）
```

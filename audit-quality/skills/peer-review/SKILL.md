# /peer-review

## 工作流

### Step 1：确定抽查样本

```
抽查年度：[YYYY]
样本选择方式：[风险导向 / 随机 / 重点项目全覆盖]
样本项目数：[N]
覆盖行业：[列表]
```

### Step 2：统一检查口径

检查项目：
- 业务承接与保持
- 独立性文件
- 风险评估和重要性
- 审计程序执行与底稿质量
- 复核记录
- 报告一致性
- 归档及时性

### Step 3：共性问题汇总

```
┌─ 共性问题汇总 ─────────────────────────────────────────┐
│ 问题类别                 │ 项目数 │ 说明                │
│ 复核留痕不完整            │ 4      │ 一级复核记录缺签字    │
│ 错报汇总与结论衔接不足     │ 2      │ 报告结论说明不充分    │
│ 关键审计事项模板化         │ 3      │ 缺乏项目特异性        │
└────────────────────────────────────────────────────────┘
```

### Step 4：输出

```
[年度]_PR_[日期].md — 同业互查报告
├── 抽样策略
├── 各项目检查结果
├── 共性问题
├── 培训与整改建议
└── 质量体系改进建议
```

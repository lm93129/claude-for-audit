# /independence-check

## 工作流

### Step 1：收集基础信息

```
项目名称：[客户 + 年度]
客户类型：[上市 / 非上市 / 金融机构 / 国企 / 其他]
项目团队：[姓名列表]
已提供或拟提供的非鉴证服务：[清单]
关键管理人员关系声明：[有/无异常]
```

### Step 2：检查维度

| 维度 | 检查项 | 结果 |
|------|------|------|
| 财务利益 | 团队成员或近亲属是否持有客户股权/债权 | ✓/⚠/✗ |
| 业务关系 | 是否存在重大商业往来 | ✓/⚠/✗ |
| 人员关系 | 是否有近亲属任客户关键岗位 | ✓/⚠/✗ |
| 非鉴证服务 | 是否提供受禁服务 | ✓/⚠/✗ |
| 长期任职/轮换 | 是否触发关键合伙人轮换或冷却期 | ✓/⚠/✗ |
| 费用依赖 | 对单一客户收费依赖是否过高 | ✓/⚠/✗ |

### Step 3：结论

```
结论类型：
- ✅ 满足独立性要求
- ⚠ 满足但需采取防范措施
- ❌ 不满足，建议退出/更换团队/停止受禁服务
```

### Step 4：输出

```
[项目编号]_IND_[日期].md — 独立性检查记录
├── 基础信息
├── 六维检查结果
├── 风险点与防范措施
└── 最终结论
```

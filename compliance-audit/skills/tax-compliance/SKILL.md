# /tax-compliance

## 工作流

### Step 1：确定审计范围

```
主体范围：[公司/子公司/分支机构]
税种范围：[增值税/企业所得税/个税/印花税/附加税/其他]
期间：[YYYY 年度或指定月份]
特殊事项：[税收优惠、重组、跨境、关联交易]
```

### Step 2：基础检查

| 检查项 | 目标 |
|------|------|
| 税务登记与税种识别 | 登记完整、税种适用正确 |
| 申报及时性 | 是否逾期申报、漏报 |
| 发票管理 | 开票、收票、作废、红字是否合规 |
| 税会差异 | 企业所得税纳税调整是否合理 |
| 优惠政策 | 适用条件是否满足、留存备查是否充分 |
| 关联交易 | 定价、同期资料、申报是否完整 |

### Step 3：异常扫描

```
┌─ 税务异常扫描 ───────────────────────────────────────────┐
│ 异常类型                    │ 数量/金额 | 备注              │
│ 增值税进项发票异常           │ 6 张      │ 抬头/税号不一致    │
│ 销项开票滞后                 │ ¥120万    │ 收入确认与开票错期 │
│ 招待费税前扣除超限           │ ¥35万     │ 需纳税调增         │
│ 高新技术企业优惠证据不足      │ 1 项      │ 缺留存备查材料     │
└────────────────────────────────────────────────────────┘
```

### Step 4：输出

```
[项目编号]_TAX_[日期].md — 税务合规审计报告
├── 税种与期间范围
├── 基础合规检查结果
├── 异常事项与风险评估
├── 补税/处罚风险估计（如适用）
└── 整改建议与优先级
```

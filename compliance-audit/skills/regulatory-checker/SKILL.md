# /regulatory-checker

## 工作流

### Step 1：明确场景

```
业务场景：[如：APP 收集用户信息 / 采购招待 / 跨境付款 / 发票管理 / 数据出境]
涉及地域：[中国大陆 / 跨境 / 特定省市]
涉及主体：[客户 / 员工 / 供应商 / 经销商 / 用户]
涉及数据或交易类型：[个人信息 / 敏感个人信息 / 资金 / 发票 / 合同]
```

### Step 2：识别适用规则

按层级输出：

| 层级 | 规则类型 | 示例 |
|------|---------|------|
| 法律 | 全国性法律 | 《个人信息保护法》《数据安全法》《税收征管法》 |
| 行政法规/部门规章 | 监管部门规则 | 网信办规定、税务总局公告、市场监管规定 |
| 国家/行业标准 | 推荐性/强制性要求 | 数据安全相关国家标准 |
| 公司内部制度 | 内部约束 | 个人信息管理制度、采购制度 |

### Step 3：义务矩阵

```
┌─ 法规适用义务矩阵 ───────────────────────────────────────┐
│ 要求项                 │ 法规依据             │ 适用性 │ 当前状态 │
│ 取得明示同意            │ 个保法第13条         │ 适用   │ 已部分落实 │
│ 最小必要原则            │ 个保法第6条          │ 适用   │ 需验证    │
│ 敏感个人信息单独同意     │ 个保法第29条         │ 视场景  │ 未见证据  │
│ 数据分类分级            │ 数据安全法相关要求    │ 适用   │ 制度存在  │
│ 礼品招待审批与登记       │ 廉洁制度/反不正当竞争 │ 适用   │ 执行待验证 │
└─────────────────────────────────────────────────────────┘
```

### Step 4：输出

```
[项目编号]_REG_[日期].md — 法规适用清单
├── 场景描述
├── 适用法规分层列表
├── 义务矩阵
├── 对现有制度/流程的影响点
└── 建议后续：进入对应专项技能审查
```

# /sox-like

## 工作流

### Step 1：界定评价范围

```
评价目标：[财务报告内控 / 关键业务内控 / 专项内控]
范围：[总部 / 子公司 / 流程模块]
关键流程：[收入、采购、资金、总账结账、固定资产、IT 一般控制]
```

### Step 2：控制框架

使用 COSO 五要素作为主框架：
- 控制环境
- 风险评估
- 控制活动
- 信息与沟通
- 监督活动

### Step 3：控制测试与缺陷分类

| 控制点 | 设计有效性 | 执行有效性 | 缺陷等级 | 备注 |
|------|-----------|-----------|---------|------|
| 月末结账复核 | 有效 | 部分无证据 | 🟠 重要缺陷 | 缺复核留痕 |
| 系统权限定期复核 | 缺失 | — | 🔴 重大缺陷 | 高风险 |
| 银行余额调节 | 有效 | 有效 | — | 正常 |

### Step 4：缺陷判定逻辑

- **重大缺陷**：合理可能导致重大错报未被及时防止或发现
- **重要缺陷**：严重性低于重大缺陷，但值得治理层关注
- **一般缺陷**：需要改进但影响可控

### Step 5：输出

```
[项目编号]_ICFR_[日期].md — 内控合规评价报告
├── 范围与框架
├── COSO 五要素评价
├── 控制测试结果
├── 缺陷分类与依据
└── 整改路线图
```

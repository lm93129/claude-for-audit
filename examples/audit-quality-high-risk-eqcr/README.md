# 示例案例：高风险项目 EQCR / 质量复核

## 场景简介

项目类型：高风险法定审计项目（虚构）
主题：项目质量控制复核（EQCR）与底稿质控
目标：演示 `audit-quality` 如何围绕独立性、重大判断、关键审计事项、底稿完整性和报告一致性，对一个高风险项目实施质量复核闭环。

## 场景背景（示例）

- 客户类型：拟上市制造业集团
- 业务特征：收入增长较快、并购整合频繁、存货和应收账款余额较高
- 审计意见目标：标准无保留意见（初步）
- 项目被列为高风险项目，触发 EQCR
- 关键审计事项初步识别为：
  - 收入确认
  - 应收账款坏账准备
  - 存货跌价准备

## 为什么这个案例是“高风险”

符合高风险触发条件中的多项：
- 拟上市/资本市场敏感项目
- 高增长与年末收入集中
- 重大会计估计较多
- 新增并购主体，报表合并复杂
- 项目组中有新晋经理首次担任项目经理

## 示例输入信息

### 1. 项目基本信息

| 项目 | 内容 |
|------|------|
| 客户 | 华北先进制造股份有限公司（虚构） |
| 项目级别 | 高风险项目 |
| 审计年度 | 2024 年 |
| 报告意见（初步） | 无保留 |
| 是否触发 EQCR | 是 |
| 复核阶段 | 报告签发前 |

### 2. 项目组与独立性线索

- 项目合伙人近 6 年持续负责该客户
- 审计经理去年曾参与客户内部控制咨询项目
- 团队已签独立性确认，但其中 1 人的年度更新日期过期
- 事务所向客户还提供了税务合规咨询服务

这些线索可用于 `independence-check` 演示：
- 长期任职/轮换风险
- 非鉴证服务边界
- 独立性确认的及时性和完整性

### 3. 重大判断线索

- 收入确认：第四季度收入占全年 43%
- 坏账准备：坏账模型参数较去年变更
- 存货跌价：部分库存周转明显放缓，但跌价计提偏保守
- 持续经营：暂无重大不确定性，但融资条款趋紧

这些线索可用于 `engagement-review` 检查：
- 风险评估是否与后续程序一致
- 重大判断是否有充分记录
- KAM 是否与底稿和报告一致

### 4. 底稿质量线索

- 部分工作底稿仅有编制人签字，无一级复核签字
- 错报汇总表与报告附注中的金额有 1 处不一致
- 审计报告初稿中的 KAM 表述与底稿备忘录不完全一致
- 部分关键支持文件未统一索引编号

这些线索可用于 `documentation-qc` 演示：
- 留痕完整性
- 勾稽一致性
- 报告与底稿一致性
- 可追溯性

### 5. 职业道德线索

- 项目组在高强度赶工阶段，部分底稿复核存在“先签后补”的倾向
- 管理层对调整分录较为敏感，多次要求“尽量别调太多”
- 项目组新人对保密边界理解不足，曾将客户报表截图发到大群讨论

这些线索可用于 `ethics-screener` 演示：
- 客观性受客户压力影响
- 专业谨慎不足
- 保密要求执行不充分

## 预期演示输出

### 1. `independence-check`

预期重点：
- ⚠ 项目合伙人轮换/长期任职关注
- ⚠ 非鉴证服务边界复核
- ⚠ 独立性年度确认过期
- 结论可能为：
  - 满足但需采取防范措施
  - 或需进一步升级评估

### 2. `engagement-review`

预期重点：
- 收入确认和坏账准备作为重大判断重点复核
- KAM 的选择和披露逻辑应与底稿证据一致
- 无保留意见是否有充分支持
- 输出很可能包括若干“有条件通过/补充后通过”的整改项

### 3. `documentation-qc`

预期会发现：
- 复核签字不完整
- 底稿索引不规范
- 报告与错报汇总勾稽不一致
- KAM 文字与底稿备忘录不一致

### 4. `ethics-screener`

预期会识别：
- 客户压力对客观性的潜在影响
- 赶工状态下职业谨慎不足
- 保密纪律执行风险

## 可能形成的质量结论

### 重大/重要质量问题候选

1. 🔴 关键报告结论与底稿勾稽不一致
   - 可能直接影响报告可靠性

2. 🟠 高风险项目的复核留痕不完整
   - 影响项目复核可验证性

3. 🟠 独立性确认管理不严格
   - 可能影响合规性和项目继续承接判断

4. 🟡 KAM 表述模板化或与项目事实衔接不足
   - 影响报告质量，但可通过补充改善

## 推荐整改方向

1. 对高风险项目设置更严格的签发前清单
2. 将错报汇总表、KAM 备忘录和报告初稿纳入统一勾稽复核
3. 对独立性年度确认设置自动提醒与锁定机制
4. 对项目组开展保密和职业道德专项培训
5. 对“先签后补”现象实施零容忍纠正

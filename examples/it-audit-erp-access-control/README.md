# 示例案例：ERP 权限与访问控制 IT 审计

## 场景简介

被审计对象：集团 ERP 系统（虚构）
主题：权限与访问控制专项 IT 审计
目标：演示 `it-audit` 如何围绕 ERP 权限体系完成一条从环境了解、一般控制检查到 CAATT 分析的最小闭环。

## 场景背景（示例）

- 集团使用统一 ERP，覆盖总账、应收、应付、存货、固定资产、采购、销售等模块
- ERP 用户约 680 人，覆盖总部与 8 家子公司
- 权限由业务部门发起、IT 运维配置、应用管理员复核
- 半年前刚完成一次岗位调整，期间存在多名人员跨岗
- 近期内审发现：
  - 个别离职员工账号未及时禁用
  - 高权限角色数量偏多
  - 同一用户可能同时拥有录入和审批权限
  - 紧急提权流程缺少事后复核

### 关注点

这个案例主要聚焦：
- 用户全生命周期管理（新增/变更/离职）
- 高权限账号管理
- 不相容职责（SoD）冲突
- 紧急权限提升与复核
- 权限定期复核

## 示例输入信息

### 1. 系统环境概况

| 项目 | 内容 |
|------|------|
| 系统名称 | 集团 ERP |
| 部署模式 | 私有云 |
| 关键模块 | 总账、采购、应付、销售、库存 |
| 账号总数 | 680 |
| 高权限账号数 | 42 |
| 最近一次权限复核 | 9 个月前 |
| 最近一次岗位调整 | 6 个月前 |

### 2. 权限管理流程（示例）

1. 业务部门提交权限申请
2. 部门负责人审批
3. IT 运维配置角色
4. 应用管理员确认生效
5. 人员离职由 HR 通知 IT 禁用账号

### 3. 已知问题线索

- 680 个账号中，发现 11 个离职员工账号仍为启用状态
- 42 个高权限账号中，14 个未见正式审批记录
- 26 个用户存在潜在 SoD 冲突，例如：
  - 采购订单创建 + 采购订单审批
  - 应付发票录入 + 付款审批
  - 凭证录入 + 凭证审核
- 紧急提权共 19 次，其中 12 次没有事后复核证据
- 上次完整权限复核距离现在已超过 9 个月

### 4. 可直接喂给 `gc-attestation` 的检查重点

- 权限申请审批链是否完整
- 高权限账号是否基于岗位最小化配置
- 离职禁用是否及时
- 权限定期复核是否执行
- 紧急提权是否有时限和回收机制

## 预期演示输出

### 1. `gc-attestation`

大概率输出：
- 访问控制领域被评为重点或高风险模块
- 发现离职禁用控制失效
- 发现高权限账号审批留痕不足
- 发现紧急提权流程执行不到位

### 2. `application-control`

虽然本案例以权限为主，但仍可补充：
- 审批流是否在系统内强制执行
- 是否允许绕过审批直接记账/付款
- 关键业务动作是否记录审计日志

### 3. `caatt`

大概率输出：
- 离职未禁用账号清单
- SoD 冲突清单
- 高权限账号异常清单
- 紧急提权未回收/未复核清单

## 可能形成的缺陷结论

### 重大/重要问题候选

1. 🔴 高权限账号审批缺失
   - 可能导致未经授权的关键交易操作和数据访问

2. 🟠 离职账号未及时禁用
   - 可能导致离职人员继续访问敏感系统

3. 🟠 SoD 冲突未识别和缓解
   - 可能导致舞弊和错误无法被及时防止或发现

4. 🟡 权限定期复核周期过长
   - 影响权限管理的持续有效性

## 推荐整改方向

1. 建立 HR 离职与 IAM/ERP 自动联动禁用机制
2. 清理高权限账号并建立审批台账
3. 建立 SoD 冲突规则库和例外审批机制
4. 对紧急提权设置自动到期与事后复核要求
5. 将权限复核周期从 9 个月缩短为季度或半年度

# 示例案例：采购回扣举报调查

## 场景简介

案件类型：采购回扣 / 商业贿赂怀疑（虚构）
触发方式：匿名举报
目标：演示 `forensic-audit` 如何从举报分流、红旗扫描、交易分析到电子证据提取形成一条调查闭环。

## 场景背景（示例）

- 举报人称：采购经理长期指定某几家供应商，并通过咨询费、返点或个人账户收款方式收取回扣
- 涉及部门：采购部、财务部、供应商管理岗
- 涉及期间：2024 年 7 月至 2025 年 1 月
- 初步线索：
  - 单一来源采购显著增加
  - 新增供应商集中出现在同一季度
  - 某些咨询服务费支出缺少工作成果证明
  - 采购经理与其中一家供应商法人存在社交关系传闻

## 演示路径

### 标准路径

1. `/forensic-audit:cold-start-interview`
2. `/forensic-audit:whistleblower-triage`
3. `/forensic-audit:red-flags`
4. `/forensic-audit:transaction-analysis`
5. `/forensic-audit:data-forensics`
6. （可选）`/forensic-audit:benford-analysis`

## 示例输入信息

### 1. 举报内容摘要（可直接喂给 whistleblower-triage）

> 举报采购经理张某在过去半年内多次绕过比价流程，指定 3 家供应商采购办公设备和咨询服务。举报人怀疑张某通过个人亲属控制的咨询公司收取回扣。部分供应商报价明显偏高，且咨询服务无成果交付。请核查 2024 年 7 月至 2025 年 1 月期间的采购和付款。

### 2. 交易线索

| 线索 | 描述 |
|------|------|
| 单一来源采购占比 | 从上年 15% 升至本期 46% |
| 新增供应商 | 6 家，其中 3 家在 2 个月内集中准入 |
| 大额咨询费 | 4 笔，共计 ¥180 万，缺工作成果证明 |
| 付款路径 | 部分款项付至供应商后 3 日内出现个人账户转账传闻 |
| 采购价格 | 某批设备单价高于市场均价 25% |

### 3. 可能的红旗（预期）

- 新增交易对手且交易量大
- 单一来源采购异常集中
- 咨询费缺少商业实质
- 交易对手关系可疑
- 价格偏离市场公允水平
- 付款和验收留痕不足

### 4. 电子数据取证线索

- 采购经理在年末多个晚上登录 ERP 并导出采购明细
- 共享盘中存在一份 `consulting_fee_split.xlsx`
- 某次付款审批后，聊天记录提到“按约定处理”
- 供应商准入资料中有相似联系人手机号

## 预期演示输出

### 1. `whistleblower-triage`

大概率结果：
- 可信度：中高
- 紧急性：高（涉及资金流出、证据可能灭失）
- 影响：中高
- 综合分类：🔴 RED

### 2. `red-flags`

重点风险领域可能被标记为：
- 交易行为红旗：高
- 管理层/关键人员行为红旗：中高
- 内控红旗：中高

### 3. `transaction-analysis`

预期发现：
- 咨询费集中支付给少数供应商
- 单一来源采购异常偏高
- 新增供应商之间存在地址/联系人/控制关系交集
- 高价采购和异常付款路径交叉重叠

### 4. `data-forensics`

预期发现：
- 非工作时间导出采购明细
- 共享盘中发现拆分回款或分配测算文件
- 日志中存在敏感文件访问或删除记录

### 5. `benford-analysis`（可选）

适合对咨询费、报销、采购金额做快速异常分布检测，作为补充信号。

## 推荐的后续行动

1. 立即固定采购、付款、审批和聊天记录相关证据
2. 对 3 家重点供应商做工商穿透与关联核查
3. 穿透大额咨询费的合同、验收和付款路径
4. 必要时临时收紧相关岗位权限
5. 将调查结果与反商业贿赂合规审计联动

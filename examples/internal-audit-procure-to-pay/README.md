# 示例案例：采购到付款流程内部审计

## 场景简介

被审计对象：集团采购中心 + 共享财务中心
流程：采购到付款（Procure-to-Pay, P2P）
目标：演示 `internal-audit` 如何完成一条从审计方案到整改跟踪的主链路。

## 场景背景（示例）

- 集团有 6 家子公司，共用一套 ERP
- 采购分为原材料采购、设备采购、办公采购三类
- 采购申请在 OA 发起，采购订单在 ERP 创建，付款审批在 OA 完成
- 近一年投诉点：
  - 单一来源采购偏多
  - 比价记录缺失
  - 部分付款先于入库
  - 供应商准入资料不完整

## 示例输入信息

### 1. 流程概览

流程步骤：
1. 需求部门发起采购申请（OA）
2. 部门负责人审批
3. 采购部询价/比价
4. 采购员在 ERP 建立采购订单
5. 采购经理审批采购订单
6. 到货验收入库
7. 财务收到发票并匹配订单/入库单
8. 付款申请审批
9. 出纳付款
10. 会计记账

### 2. 已知控制设计

- 超过 50 万采购需二级审批
- 采购订单和付款申请分属不同岗位
- ERP 支持三方匹配（PO / 入库 / 发票）
- 供应商准入需要营业执照、开户资料和合规声明

### 3. 已知问题线索

- 25 笔样本中有 3 笔采购订单缺二级审批
- 40 笔付款中有 5 笔付款日期早于入库日期
- 15 家新增供应商中有 4 家资料不全
- 某月单一来源采购占比达到 48%
- 3 笔咨询服务费缺工作成果证明

### 4. 数据分析演示用指标

| 指标 | 当前值 | 参考/预期 |
|------|-------:|----------:|
| 单一来源采购占比 | 48% | <20% |
| 缺比价记录采购单数 | 12 | 0 |
| 付款早于入库的笔数 | 5 | 0 |
| 新增供应商资料缺失率 | 27% | 0% |
| 超权限审批采购单数 | 3 | 0 |

## 预期演示输出

### 1. `audit-program`

输出：
- 审计目标
- 审计范围
- 审计程序矩阵
- 时间与资源预算

### 2. `process-mapping`

输出：
- 采购到付款的步骤表
- 关键控制点（预算检查、二级审批、三方匹配、付款双人控制）
- 初步风险点

### 3. `risk-control-matrix`

输出：
- 风险 ↔ 控制 映射关系
- 控制设计有效/无效判断
- 哪些控制需执行测试

### 4. `data-analytics`

输出：
- 单一来源采购异常
- 付款时点异常
- 新增供应商资料缺失异常
- 咨询费工作成果证明缺失异常

### 5. `finding-drafting`

很可能形成的发现包括：
- 🟠 采购审批执行不到位
- 🔴 供应商准入控制薄弱
- 🟠 三方匹配例外未及时阻断
- 🟡 咨询服务验收留痕不足

### 6. `report-drafting`

输出：
- 执行摘要
- 发现分级
- 控制评价
- 整改建议汇总

### 7. `remediation-tracking`

输出：
- 整改责任人
- 计划完成日期
- 验证测试要求
- 关闭标准

## 推荐的整改建议方向

可作为 demo 中的参考答案：

1. 建立审批代理机制，避免出差造成二级审批缺失
2. 对 ERP 的三方匹配例外设置阻断规则，而非仅提醒
3. 新增供应商必须资料齐全后才允许下单
4. 单一来源采购建立例外审批和季度复盘机制
5. 咨询服务费付款前必须附工作成果和验收记录

# /review-notes

## 重要：先加载 Practice Profile

读取 `~/.claude/plugins/config/claude-for-audit/statutory-audit/CLAUDE.md` → 三级复核结构：

| 复核层级 | 由谁执行 |
|---------|---------|
| 一级复核（项目组内互查） | [高级审计员/审计经理] |
| 二级复核（项目负责人） | [项目经理/签字CPA] |
| 三级复核（质量控制复核人） | [质量控制合伙人] |

## 工作流

### Step 1：选择复核层

| 参数 | 层级 | 执行人 | 复核内容 |
|------|------|-------|---------|
| `--level1` | 一级复核 | 项目组内部 | 底稿完整性、数字勾稽、程序执行 |
| `--level2` | 二级复核 | 项目负责人 | 重大判断、未更正错报、审计策略 |
| `--level3` | 三级复核 | 质控复核人 | 独立性、关键审计事项、报告措辞 |
| （默认） | → 交互选择 | | |

### Step 2：复核清单

**一级复核清单：**

```
□ 所有审计程序已按审计计划执行并记录
□ 底稿支持性文件齐全（合同、凭证、确认函等）
□ 数字勾稽关系正确（试算表 ↔ 底稿 ↔ 报告）
□ 函证回函差异已处理或已执行替代程序
□ 抽样记录完整，结论清晰
□ 底稿头信息完整（编制人、日期、编码）
```

**二级复核清单：**

```
□ 风险评估结论合理，与后续程序一致
□ 重要性水平运用恰当
□ 关键审计事项的处理和披露适当
□ 未更正错报汇总经管理层确认
□ 重大判断有充分依据
□ 期后事项和持续经营评估充分
□ 审计意见类型正确
```

**三级复核清单：**

```
□ 项目组独立性确认函已签署
□ 关键审计事项的确定和描述适当
□ 与治理层沟通记录完整
□ 审计报告措辞准确
□ 质量控制符合事务所内控制度
□ 项目组对重大问题的处理恰当
□ 底稿归档准备就绪
```

### Step 3：创建复核意见

```
┌─ 复核意见 ───────────────────────────────────────┐
│ 底稿编号：[如 A1-01-01 审计计划]                      │
│ 复核层级：一级复核                                    │
│ 复核人：[姓名]    日期：[日期]                         │
│                                                    │
│ 意见：                                                │
│ 1. 风险评估中对关联方交易的识别程序未完全记录，请补   │
│    充已执行的相关询问程序。                             │
│ 2. 第 3 页中列示的营业收入增长分析缺少与行业增长率     │
│    的对比，建议补充。                                  │
│                                                    │
│ 结论：[需整改后重新提交 / 通过 / 通过但要补充说明]       │
└──────────────────────────────────────────────────────┘
```

### Step 5：重大问题讨论记录

对于复核中发现的需要讨论的重大问题，创建讨论备忘录：

```
## 重大问题讨论记录

### 问题描述
[复核人在复核[底稿编号]时发现……]

### 讨论参与人
[签字CPA、质量控制复核人、项目组成员]

### 讨论日期
[日期]

### 讨论过程
1. [观点A]
2. [观点B]
3. [最终结论]

### 最终结论
[经讨论后决定……]

### 影响
- 对审计策略的影响：[无/扩大程序范围/修改审计意见]
- 对审计报告的影响：[无/修改措辞]
```

### Step 6：输出

```
[底稿编码]-RV-01 复核意见汇总.xlsx
  → Sheet 1：各底稿复核状态
  → Sheet 2：待整改事项清单（按期限排序）
  → Sheet 3：已完成复核记录

[底稿编码]-RV-02 重大问题讨论记录.xlsx （仅在重大问题存在时生成）
```

### 复核完成后

当三级复核全部完成后：

> ✅ 三级复核完成。已具备签发审计报告的条件。
> 下一步：`/statutory-audit:report-drafting` — 生成最终审计报告草稿。

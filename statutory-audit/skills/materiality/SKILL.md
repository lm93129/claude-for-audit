# /materiality

## 重要：先加载 Practice Profile

读取 `~/.claude/plugins/config/claude-for-audit/statutory-audit/CLAUDE.md` → 审计方法论 → 重要性水平：

- 基准选择：`[营业收入 / 净利润 / 总资产 / 净资产]`
- 基准占比：`[N%]`
- 实际执行重要性比例：`[50% / 75%]`
- 明显微小错报阈值：`[N% of PM]`

如配置缺失 → 返回错误："请先完成 cold-start-interview，或在 customize 中设置重要性偏好。"

## 工作流

### Step 1：获取基准数据

向用户收集最近一期经审计或未经审计的财务数据：

| 项目 | 本年数（元） | 上年数（元） |
|------|------------|------------|
| 营业收入 | | |
| 净利润 | | |
| 总资产 | | |
| 净资产 | | |

如连接了企业 ERP MCP → 自动获取试算表数据并填入。

### Step 2：计算重要性水平

```
基准指标：营业收入       基准值：¥100,000,000
适用比例：5%
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
计划重要性（PM）          = ¥5,000,000
实际执行重要性（TE）      = PM × 75% = ¥3,750,000
                                                         明显微小错报阈值（SAD） = PM × 5%  = ¥250,000
```

输出格式：按 Practice Profile 中配置的格式。

### Step 3：合理性评估

验证所选基准对当前被审计单位是否适用：
- 如选择净利润但近三年盈亏波动大 → 建议改用营业收入
- 如选择总资产但属于轻资产行业 → 建议改用营业收入
- 如被审计单位属于新设企业 → 建议考虑总资产

如发现不匹配 → 标记【建议调整】并说明理由。

### Step 4：输出底稿

写入底稿文件：

```
[底稿编码]-MAT-01 重要性水平计算表.xlsx
Sheet 1：计算表（公式链接）
Sheet 2：调整记录
Sheet 3：复核意见
```

前置工作底稿头和复核状态标注。

# /report-drafting

## 重要：先加载 Practice Profile

读取 `~/.claude/plugins/config/claude-for-audit/statutory-audit/CLAUDE.md` → 审计报告：
- 报告格式：[标准格式 / 关键审计事项格式 / 注册制格式]
- 报告语言：[简体中文/英文/双语]
- 报告草稿会签：[需要/不需要]
- 报告签字方式：[双签字/单签字]

## 前序要求

本技能运行前需已完成：
- ✅ 实质性程序 → 错报已汇总
- ✅ 三级复核 → 复核已通过
- ✅ 管理层书面声明已获取
- ✅ 与治理层的沟通已完成

**如未完成以上任一项 → 提示先完成。**

## 工作流

### Step 1：加载审计结果

从各技能的输出自动收集：

```
错报汇总：
  事实错报合计：¥350,000
  判断错报合计：¥150,000
  推断错报合计：¥180,000
  未更正错报合计：¥500,000

重要性水平：
  计划重要性（PM）：¥5,000,000
  实际执行重要性（TE）：¥3,750,000
  明显微小错报（SAD）：¥250,000

关键审计事项：[来自 risk-assessment 的预期 / 新增事项]
持续经营评估：[不存在重大不确定性 / 存在重大不确定性]
期后事项：[无调整事项 / 有调整事项]
```

### Step 2：确定审计意见类型

根据未更正错报的性质和金额判断：

```
未更正错报合计 ¥500,000 VS 计划重要性 ¥5,000,000
  → 未超过 PM → 初步判断：无保留意见 ✓

检查：
□ 未更正错报是否广泛影响财务报表？→ 否
□ 是否存在审计范围受限？→ 否
□ 是否存在重大不确定性已充分披露？→ 是
□ 管理层书面声明已获取？→ 是

判断结论：无保留意见 ✅
```

**如果遇到以下情况，提示用户并建议对应意见类型：**

| 情况 | 意见类型 |
|------|---------|
| 错报超过 PM 但不广泛 | 保留意见 |
| 错报超过 PM 且广泛 | 否定意见 |
| 审计范围受限但不广泛 | 保留意见 |
| 审计范围受限且广泛 | 无法表示意见 |
| 持续经营存在重大不确定性但已充分披露 | 无保留+强调事项段 |
| 期后事项重大但已正确处理 | 无保留+其他事项段 |

### Step 4：审计报告会签

如 Practice Profile 中报告草稿需要会签：

```
会签清单：
□ 项目合伙人：[姓名/日期]
□ 质量控制复核人：[姓名/日期]
□ 风险管理部（如适用）：[姓名/日期]
□ 首席合伙人（如适用）：[姓名/日期]
```

### Step 5：输出

```
[底稿编码]-REPORT-01 审计报告草稿.docx
  → 标准文字格式，可直接用于最终排版

[底稿编码]-REPORT-02 审计报告签署确认单.xlsx
  → 签字记录、会签跟踪

[底稿编码]-WP-99 审计完成备忘录.xlsx
  → 最后一页底稿，记录从计划到报告的全过程关键节点
```

### 签发前最终检查清单

```
□ 审计意见类型正确
□ 关键审计事项恰当
□ 审计报告日期为现场工作完成日
□ 所有需要的签字已签署
□ 底稿已在 60 天内归档（或已排期）
□ 管理层书面声明已归档
□ 与治理层沟通记录完备
□ 独立性确认函已签署
```

### 完成后指引

> ✅ 审计报告草稿已完成。请按以下步骤推进签发：
> 1. 将报告草稿提交事务所内部审批
> 2. 获取管理层书面声明签字
> 3. 与治理层沟通最终审计意见
> 4. 取得所有签字后签发正式报告
> 5. 完成底稿归档（审计报告日后 60 天内）

# /cold-start-interview

首次运行写入 `~/.claude/plugins/config/claude-for-audit/statutory-audit/CLAUDE.md`；后续 `--redo` 重新采访并在覆盖前显示 diff。

## 检查当前状态

读取 `~/.claude/plugins/config/claude-for-audit/statutory-audit/CLAUDE.md`：
- **不存在** → 开始采访。
- **包含 [PLACEHOLDER] 或 [事务所名称]** → 询问是重头开始还是从占位符处继续。
- **已完整填写且未传 --redo** → 告知已配置，询问是否需要重跑。

## 采访脚本

### Part 0：连接器探测

检查 `.mcp.json` 中声明的连接器哪些实际可用。只报告实际调用成功的（标记 ✓），已配置但未测试的标记 ⚪。不把声明当成可用。

### Part 1：我们是谁

1. 你的事务所名称是什么？
2. 执业资质类型？*（证券期货 / 一般企业 / 金融审计 / 其他）*
3. 执业CPA人数？质量控制负责人是谁？
4. 年审项目大约多少个？项目组通常几人？
5. 使用什么审计软件？*（鼎信诺 / 用友审计 / 审易 / 自研 / 无）*
6. 底稿是全电子化还是需要打印纸质存档？

### Part 2：审计方法论

**重要性水平：**
1. 你通常用什么基准计算计划重要性？*（营业收入总额 / 净利润 / 总资产 / 净资产）*
2. 百分比多少？
3. 实际执行重要性用计划重要性的 50% 还是 75%？

**审计策略：**
1. 通常是常规策略还是扩大策略？
2. 依赖内部控制的程度？（高/中/低）
3. 抽样偏好？（统计/非统计/货币单位）

**函证：**
1. 银行存款是否 100% 函证？
2. 应收/应付账款函证比例目标？
3. 回函率不理想时的替代程序偏好？

### Part 3：底稿与复核

1. 底稿编码规则？（有无事务所内部标准）
2. 三级复核各由谁执行？
3. 审计报告格式？（标准格式 / 关键审计事项格式 / 注册制格式）

### Part 4：行业与客户

1. 被审计单位属于什么行业？
2. 适用什么会计准则？（CAS / IFRS / 企业会计制度）
3. 本年预期有哪些关键审计事项？

### Part 5：看门文件

要求提供种子材料（越多越好）：
- 上年审计报告副本（脱敏）
- 上年审计底稿 2-3 份（脱敏）
- 事务所内部审计方法论手册（如有）
- 重要性水平计算表样本
- 审计报告模板

从这些文件中提取真实的方法论证据，并核验采访中所述与实践是否一致。

## 写出 Practice Profile

按模板结构写入 `~/.claude/plugins/config/claude-for-audit/statutory-audit/CLAUDE.md`，使用受访者的原话。

同时检查 `~/.claude/plugins/config/claude-for-audit/company-profile.md`：
- 存在：读取后确认事务所信息，跳过公司问题
- 不存在：创建并写入事务所级别信息（所有插件共享）

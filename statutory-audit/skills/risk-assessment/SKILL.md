# /risk-assessment

## 重要：先加载 Practice Profile

读取 `~/.claude/plugins/config/claude-for-audit/statutory-audit/CLAUDE.md` → 审计方法论：
- 被审计单位行业
- 年度关键审计事项预期
- 审计策略偏好（常规/扩大/简化）
- 适用的会计准则

## 前置工作

如连接了企查查/天眼查 MCP，自动获取被审计单位信息：
- 工商信息（注册资本、成立日期、股权结构）
- 行政处罚和涉诉记录
- 对外投资和关联方
- 主要人员变更情况

如连接了企业 ERP MCP，自动获取本期预审数据。

## 工作流

### Step 1：了解被审计单位及其环境

**行业状况：**
```
行业分类：[制造业/金融/互联网/房地产/……]
行业监管环境：[强监管/一般/宽松]
行业竞争格局：[寡头/充分竞争/新兴]
行业特有财务指标：[毛利率区间、周转率基准]
```

**法律与监管环境：**
```
适用的会计准则：[CAS / IFRS]
适用税法：[企业所得税 25%/高新15%/小微企业]
行业特定法规：[如金融企业需遵循的监管要求]
```

**被审计单位性质：**
```
所有权结构：[上市/非上市/国企/外资]
治理结构：[董事会/监事会设置]
主营业务：[描述]
投资与筹资活动：[重大投资计划/融资安排]
```

### Step 2：了解被审计单位的内部控制

**控制环境：**
```
治理层诚信与道德价值观：[初步印象/需关注]
管理层哲学与经营风格：[稳健/激进]
职责分离：[合理/存在不相容岗位未分离]
```

**被审计单位的风险评估过程：**
```
正式的风险评估：[存在/不存在/未知]
舞弊风险评估：[已执行/未执行]
```

**信息系统与沟通：**
```
财务系统：[用U8/金蝶K3/SAP/Oracle/自研]
业务流程信息化程度：[高/中/低]
```

**控制活动：**
```
授权审批制度：[完善/基本完善/薄弱]
财产保护：[有制度/无制度]
绩效考核：[与财务报告挂钩/不挂钩]
```

**对内部控制的初步评价：**
```
控制环境：[有效/需关注/薄弱]
是否计划依赖内部控制：[是/否]（来自 Practice Profile 或本次判断）
```

### Step 3：识别和评估重大错报风险

**固有风险（在考虑控制前）：**

| 认定层次 | 科目 | 风险描述 | 风险等级 | 影响的重要类别 |
|---------|------|---------|---------|-------------|
| 存在/发生 | 营业收入 | 管理层为达成业绩指标虚构收入 | 高（舞弊风险推定）| 利润表 |
| 完整性 | 负债 | 未入账负债 | 中 | 资产负债表 |
| 计价/分摊 | 应收账款 | 坏账准备计提不充分 | 高（本行业常见）| 资产负债表 |
| 权利/义务 | 存货 | 代管/寄售存货权属不清 | 低 | 资产负债表 |

**舞弊风险因素（CSA 1141 号要求）：**
```
管理层压力：[业绩对赌/上市承诺/融资对赌]
管理层机会：[内控薄弱/治理层监督不足]
管理层态度：[对财务报告态度激进/过往调整较多]
```

### Step 4：制定初步审计策略

| 科目 | 风险等级 | 拟实施的程序 | 计划覆盖时间 |
|------|---------|------------|------------|
| 营业收入 | 高（舞弊推定）| 实质性程序（细节测试+分析） | 全年 |
| 应收账款 | 高 | 函证+替代+账龄分析 | 期末 |
| 存货 | 中 | 监盘+计价测试 | 期末 |
| 固定资产 | 低 | 分析性复核+抽盘 | 年末 |

### Step 5：输出风险评估底稿

```
[底稿编码]-RA-01 了解被审计单位及其环境.xlsx
[底稿编码]-RA-02 了解内部控制.xlsx  
[底稿编码]-RA-03 重大错报风险评估.xlsx
[底稿编码]-RA-04 舞弊风险评估（CSA 1141）.xlsx
```

每个底稿包含复核状态标注。

### 输出的后续指引

风险评估完成后，建议下一步：
> 风险评估已完成。下一步推荐：
> 1. `/statutory-audit:materiality` — 确定重要性水平
> 2. `/statutory-audit:ic-review` — 深入执行控制测试
> 3. `/statutory-audit:substantive-testing` — 直接执行实质性程序

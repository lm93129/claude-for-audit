# /substantive-testing

## 重要：先加载 Practice Profile

读取 `~/.claude/plugins/config/claude-for-audit/statutory-audit/CLAUDE.md` → 审计方法论：
- 实质性程序策略（常规/扩大/简化）
- 实际执行重要性（TE）
- 明显微小错报阈值（SAD）

## 前序输入要求

用户需先执行或加载：
1. `/statutory-audit:risk-assessment` — 了解风险分布
2. `/statutory-audit:materiality` — 确定重要性水平
3. `/statutory-audit:ic-review` — （如有）控制测试结果

如尚未完成 → 提示先完成计划阶段工作。

## 工作流

### Step 1：明确测试科目

支持以下科目（通过参数指定）：

| 参数 | 科目 | 典型风险领域 |
|------|------|------------|
| `--revenue` | 营业收入 | 虚构收入、截止性错误 |
| `--ar` | 应收账款 | 存在性、计价（坏账准备）|
| `--inventory` | 存货 | 存在性、计价（跌价准备）、权属 |
| `--ppne` | 固定资产 | 存在性、折旧、减值 |
| `--ap` | 应付账款 | 完整性、计价 |
| `--cash` | 货币资金 | 存在性、完整性、受限资金 |
| `--loan` | 借款 | 完整性、分类、利息计提 |
| `--equity` | 所有者权益 | 实收资本、留存收益 |
| `--all` | 所有主要科目 | 期末全面测试 |
| （默认） | → 交互选择 | |

如不传参数 → 展示菜单让用户选择。


### Step 4：汇总发现的错报

```
┌─ 错报汇总 ─────────────────────────────────────────────┐
│ 序号 │ 科目     │ 描述             │ 金额(元) │ 类型   │ 调整建议  │
│ 1    │ 营业收入 │ 跨期确认收入1笔   │ 200,000   │ 事实错报 │ 借记主营业务收入 │
│ 2    │ 应收账款 │ 坏账准备计提不足   │ 150,000   │ 判断错报 │ 补提信用减值损失 │
│ …    │ …       │ …               │ …        │ …      │ …          │
│      │          │                │           │        │            │
│      │ 小计     │                │ 350,000   │        │            │
│      │ SAD门槛   │                │ 250,000   │        │            │
│      │ 是否调整  │                │           │        │             │
└─────────────────────────────────────────────────────────┘
```

错报分类：
- **事实错报**：确定存在且可量化的错报
- **判断错报**：管理层的会计估计与审计师判断的差异
- **推断错报**：抽样中发现的错报推断至总体

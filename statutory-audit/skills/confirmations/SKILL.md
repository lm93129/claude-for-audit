# /confirmations

## 重要：先加载 Practice Profile

读取 `~/.claude/plugins/config/claude-for-audit/statutory-audit/CLAUDE.md` → 函证策略。

## 工作流

### Step 1：确定函证范围

**银行存款：** 所有账户（含零余额和本年度注销户）

**往来款：** 按重要性水平筛选：
- 超过实际执行重要性的客户/供应商 → 全部函证
- 低于但合计数超过某比例的 → 分层抽样函证
- 关联方往来 → 全部函证

### Step 2：生成询证函

生成银行询证函（标准模板）和往来款询证函：

```
┌─────────────────────────────────────────┐
│          询 证 函                          │
│                                          │
│ 编号：[底稿编码]-CNF-001                   │
│ 致：[被函证单位]                           │
│                                          │
│ 本公司聘请的[事务所名称]正在对[公司名称]   │
│ 截至 202X年12月31日的财务报表进行审计……    │
└─────────────────────────────────────────┘
```

### Step 3：跟踪函证状态

| 被函证单位 | 科目 | 金额(元) | 发函日 | 回函日 | 状态 | 差异 |
|-----------|------|---------|-------|-------|------|------|
| 客户A | 应收账款 | 1,000,000 | 2025-01-20 | 2025-02-05 | ✓ 一致 | 无 |
| 客户B | 应收账款 | 500,000 | 2025-01-20 | — | ⏳ | — |
| 客户C | 应付账款 | 800,000 | 2025-01-20 | — | ⚠ 超期 | — |

### Step 4：处理回函差异

差异分析 → 归类（时间性差异/记录错误/虚构交易） → 建议调整分录。

### Step 5：替代程序

对未回函的客户执行替代程序：
1. 检查期后回款记录
2. 检查销售合同、发货单、签收单
3. 检查对应发票

### Step 6：输出底稿

```
[底稿编码]-CNF-01 函证控制表.xlsx
[底稿编码]-CNF-02 替代程序记录.xlsx
[底稿编码]-CNF-03 回函差异汇总.xlsx
```
